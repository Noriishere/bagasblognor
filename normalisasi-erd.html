Apa Itu ERD dan Normalisasi Basis Data?

Disclaimer! : Informasi yang saya sampaikan merupakan pemahaman pribadi yang didukung materi kuliah dan sumber online. Jika ada kesamaan, mohon hubungi admin.

Analogi 1: Cetak Biru (ERD)

ERD (Entity-Relationship Diagram) adalah cetak biru atau *blueprint* dari basis data kita. Sebelum membangun rumah, arsitek membuat gambar yang menunjukkan ada ruangan apa saja (Entitas), pintu penghubung antar ruangan (Relasi), dan detail tiap ruangan seperti 'jumlah jendela' (Atribut).

Dalam basis data, ERD membantu kita merancang secara visual:

  • Entitas: Objek utama (misal: 'Mahasiswa', 'MataKuliah').
  • Atribut: Detail dari entitas (misal: 'NIM' dan 'Nama' pada Mahasiswa).
  • Relasi: Hubungan antar entitas (misal: Mahasiswa 'Mengambil' MataKuliah).

ERD adalah tahap desain konseptual (ide) sebelum kita benar-benar membuat tabel.

Analogi 2: Merapikan Lemari (Normalisasi)

Normalisasi adalah proses merapikan tabel yang sudah kita buat (berdasarkan ERD) agar efisien. Bayangkan Anda punya satu lemari besar tempat menyimpan baju, celana, kaos kaki, dan sepatu (Tabel Un-normalized). Akan susah mencari barang dan memakan tempat.

Normalisasi adalah proses membagi lemari besar itu menjadi rak-rak khusus: rak sepatu, rak baju, rak celana (Tabel Normalized). Tujuannya adalah untuk mengurangi redundansi (duplikasi) data dan mencegah anomali (keanehan saat update, insert, atau delete data).

Contoh Kode: Hasil Normalisasi (SQL)

Kita "merapikan" tabel tadi dengan memecahnya menjadi 3 tabel yang lebih fokus, menggunakan kunci (PRIMARY KEY & FOREIGN KEY) sebagai penghubung. 


-- 1. Tabel Mahasiswa (Hanya data mahasiswa)
CREATE TABLE Mahasiswa (
  NIM VARCHAR(15) PRIMARY KEY,
  Nama_Lengkap VARCHAR(100)
);

-- 2. Tabel MataKuliah (Hanya data matkul)
CREATE TABLE MataKuliah (
  Kode_MK VARCHAR(10) PRIMARY KEY,
  Nama_MataKuliah VARCHAR(100),
  Dosen VARCHAR(100)
);

-- 3. Tabel KRS (Hanya berisi 'relasi')
CREATE TABLE KRS (
  ID_KRS INT PRIMARY KEY,
  NIM_Mahasiswa VARCHAR(15),
  Kode_MK_Kuliah VARCHAR(10),
  FOREIGN KEY (NIM_Mahasiswa) REFERENCES Mahasiswa(NIM),
  FOREIGN KEY (Kode_MK_Kuliah) REFERENCES MataKuliah(Kode_MK)
);
